# XAuth Omega - Integration Examples

This directory contains official implementation examples of the **XAuth Omega Protocol**. These scripts demonstrate how to securely communicate with the XAuth API to validate licenses, manage sessions, and protect your software from unauthorized use.

## 🌐 Official URLs
- **Admin Dashboard**: [https://xauth.monster](https://xauth.monster)
- **API Endpoint**: `https://api.xauth.monster`

---

## 🔒 The Omega Protocol
The protocol used in these examples is designed for high-security environments:
1.  **Handshake**: The client initializes a session and receives a unique `nonce`.
2.  **Encrypted Payload**: All validation data is sent with a cryptographically secure `HWID` (Hardware ID).
3.  **AES-GCM Validation**: The server response is encrypted using `AES-256-GCM`. The encryption key is dynamically generated by combining your `APP_SECRET` and the session `nonce`.

## 🚀 Quick Start

### 1. Requirements
Ensure you have created an application in your XAuth Dashboard and retrieved your **App ID** and **App Secret**.

### 2. Basic Implementation (Python Example)
```python
from xauth import XAuth

# Initialize the client
client = XAuth(
    app_id=1, 
    app_secret="your_very_secret_key", 
    base_url="https://api.xauth.monster"
)

# Validate a license
result = client.validate_license("XXXX-XXXX-XXXX-XXXX")

if result["success"]:
    print(f"Welcome! License expires on: {result['expiry']}")
    print(f"Message from developer: {result['broadcast']}")
else:
    print(f"Access Denied: {result['message']}")
```

## 🛡️ Security Best Practices
- **Obfuscation**: Always obfuscate your production binaries to prevent attackers from stripping the XAuth check.
- **Secret Key Security**: Never hardcode your `APP_SECRET` in a way that is easily readable in plain text.
- **Server-Side Variables**: Use the `variables` feature in XAuth to store sensitive configuration data that only gets sent to the client after a successful validation.
